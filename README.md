# Profile Purity Detector

AI-assisted fake/real profile detection for Instagram and Twitter/X profiles.

## How it works

This is a single Next.js app (deployed on Vercel) - there is no separate backend service to run or deploy.

1. **Paste a profile URL** (optional). `/api/extract` does a best-effort fetch of the public profile page and reads its Open Graph meta tags (title, description, image). Instagram/X frequently block this, rate-limit it, or don't expose enough in those tags (private accounts especially) - whatever can't be extracted, you fill in yourself on the next screen. This is not guaranteed to work and does not use any private/authenticated API.
2. **Manual entry / review.** Confirm or fill in username, bio, follower/following/post counts, verification, privacy, and optionally a profile photo.
3. **Analysis** (`/api/analyze`) combines two independent, real signals:
   - **A trained classifier** (`frontend/lib/metricsModel.ts`) - a logistic regression trained offline (see `backend/ml/train_metrics_model.py`) on a public, labeled Instagram fake-account dataset. It is deterministic: the same inputs always produce the same score, and its real held-out accuracy/AUC (not a fabricated number) ships in the response.
   - **Gemini** (`frontend/lib/gemini.ts`), which reads the bio text and photo (if provided) and returns a structured authenticity assessment + explanation. Requires a `GEMINI_API_KEY`; if it's not configured, the app falls back to the classifier alone and says so in the response rather than inventing a number.

Nothing in the scoring path uses `Math.random()` or hardcoded point values disguised as "AI."

## Running locally

```bash
cd frontend
npm install
cp .env.local .env.local.example 2>/dev/null || true  # if you haven't already, edit .env.local
npm run dev
```

Open http://localhost:3000.

### Gemini API key (optional but recommended)

Without it, analysis still works using the trained classifier alone.

1. Get a free key at https://aistudio.google.com/apikey
2. Add it to `frontend/.env.local`: `GEMINI_API_KEY=...`
3. For production, set `GEMINI_API_KEY` in your Vercel project's Environment Variables (not in a committed file).

## Retraining the classifier

`backend/` is **not deployed** - it's an offline tool you run locally whenever you want to retrain on new/better data.

```bash
cd backend
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python ml/train_metrics_model.py
```

This prints real accuracy/precision/recall/AUC on a held-out test split and writes the learned weights to `frontend/lib/metricsModel.json`, which `metricsModel.ts` uses directly (no Python needed at runtime).

Dataset: `backend/data/instagram_fake_accounts/{train,test}.csv` - a public, labeled Instagram fake/spammer/genuine-accounts dataset (11 numeric profile features: profile picture presence, username/fullname character patterns, bio length, external link, privacy, post/follower/following counts).

## Project structure

```
frontend/
  app/api/analyze/route.ts   # combines metricsModel score + Gemini into a trust score
  app/api/extract/route.ts   # best-effort URL -> profile data extraction
  lib/metricsModel.ts        # deterministic logistic-regression scorer (TS port of the trained model)
  lib/metricsModel.json      # trained weights (generated by backend/ml/train_metrics_model.py)
  lib/gemini.ts              # Gemini REST call for text/image assessment
  components/                # UI (shadcn/ui + Tailwind)
backend/
  ml/train_metrics_model.py  # offline training script, not deployed
  data/instagram_fake_accounts/  # labeled training data
```

## Known limitations

- URL extraction is best-effort only. Platforms actively limit what's visible without logging in; expect frequent fallback to manual entry, especially for follower/following/post counts on Twitter/X.
- The trained classifier's features (profile picture, username pattern, bio length, follower/following ratios, etc.) reflect what a public Instagram dataset could measure - it will generalize imperfectly to Twitter/X or to fraud patterns not represented in that dataset.
- This is a decision-support tool, not a certified fraud-detection system - false positives/negatives are expected.
